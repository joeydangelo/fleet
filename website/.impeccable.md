## Design Context

### Users
Developers and builders who use AI coding agents. They're technical, value precision over hype, and evaluate tools by what they do, not how they're marketed. They use the CLI daily and want a website that reflects the same care as the tool itself.

### Brand Personality
**Precise, capable, calm.** Fleet is a serious tool for serious work. It doesn't need to shout. The product speaks for itself through clear documentation and a polished demo.

### Aesthetic Direction
- **Visual tone**: Warm, light, editorial. Think crafted technical documentation, not SaaS marketing.
- **Palette**: Warm neutrals from the Remotion demo — `#faf9f7` (bg), `#f2f0ec` (alt bg), `#e8e5df` (chrome), `#d4d0ca` (borders). Accents: green `#7fa887`, blue `#6a9bcc`, orange `#d97757`.
- **Typography**: Instrument Serif for display headlines (distinctive, editorial — unexpected for a dev tool, signals craftsmanship). Plus Jakarta Sans for body text. Fira Code for code/commands.
- **Anti-reference**: BridgeMind.ai — dark backgrounds, gradient text, neon accents, "vibe coding" hype. Fleet is the opposite: light, warm, structured, honest.
- **Texture**: Subtle dot-grid pattern (from demo video backdrop) used sparingly as a background element.
- **Theme**: Light only. No dark mode.

### Design Principles
1. **Let the product speak** — Show the demo, list the commands, explain the lifecycle. No marketing fluff.
2. **Warm precision** — Every color, font weight, and spacing value is intentional. Warm neutrals over cold grays.
3. **Terminal authenticity** — Code blocks, command references, and status indicators echo the real CLI experience.
4. **Calm confidence** — Generous whitespace, restrained motion, clear hierarchy. Nothing fights for attention.
5. **Editorial over commercial** — Serif headlines, structured layouts, and honest copy. More technical paper than SaaS landing page.
